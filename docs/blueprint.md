# **App Name**: NEU Access

## Core Features:

- User Role Selection: A prominent landing page where users must explicitly choose between 'Visitor (User)' and 'Admin' roles.
- Google Institutional Login: Secure authentication flow utilizing Google Sign-In, exclusively restricted to institutional emails within the `@neu.edu.ph` domain.
- Role-Based Redirection: Automatic redirection of authenticated users to either a dedicated Admin Dashboard or a Visitor onboarding flow based on their chosen role.
- Admin Dashboard Placeholder: A basic, navigable interface representing the entry point for administrators post-authentication.
- Visitor Onboarding Flow Placeholder: A minimal sequence of screens indicating the starting point for visitor interactions post-authentication and role selection.
- Cloud Firestore Integration: Initial setup and connection to Cloud Firestore for robust and scalable storage of application data.

## Style Guidelines:

- Primary action color: NEU Green (#1B6E3F), representing institutional identity and professionalism.
- Background color: A very light, desaturated green (#E7F7ED), providing a clean and subtle base that harmonizes with the primary color.
- Accent color: A softer, analogous lime-green (#ACE582), used for highlighting interactive elements or key information without overpowering the primary palette.
- Body and headline font: 'Inter' (sans-serif) for a modern, objective, and highly readable appearance suitable for an institutional application.
- Utilize a consistent set of clean, professional, and easily recognizable line icons to enhance clarity and functionality across the interface.
- Employ a structured, functional, and organized layout with ample whitespace, ensuring a clear visual hierarchy for intuitive navigation and data presentation.
- Incorporate subtle and functional micro-animations for interactive elements (e.g., button presses, state changes) to provide clear feedback and enhance perceived responsiveness.
